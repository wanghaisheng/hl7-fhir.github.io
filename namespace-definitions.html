6.13.5 Resource Namespace - Formal Definitions

Formal definitions for the elements in the Namespace resource.

Namespace
DefinitionA curated namespace that issues unique symbols within that namespace for the identification of concepts, people, devices, etc. Represents a "System" used within the Identifier and Coding data types.
Control1..1
InvariantsDefined on this element
Inv-1: Root systems cannot have uuid or sid identifiers (xpath: not(f:type/@value='root' and f:uniqueId/f:type=('uuid', 'ruid')))
Inv-2: Can't have more than one preferred identifier for a type (xpath: not(exists(for $type in distinct-values(f:uniqueId/f:type) return if (count(f:uniqueId[f:type/@value=$type and f:primary/@value='true'])>1) then $type else ())))
Namespace.type
DefinitionIndicates the purpose for the namespace - what kinds of things does it make unique?.
Control1..1
BindingNamespaceType: Identifies the purpose of the namespace (see http://hl7.org/fhir/namespace-type for values)
Typecode
Namespace.label
DefinitionThe name of this particular identifier or code system.
Control1..1
Typestring
CommentsThe"symbolic name" would be captured as an extension.
Namespace.status
DefinitionIndicates whether the namespace is "ready for use" or not.
Control1..1
BindingNamespaceStatus: Indicates whether the namespace should be used (see http://hl7.org/fhir/namespace-status for values)
Typecode
Namespace.country
DefinitionIf present, indicates that the identifier or code system is principally intended for use or applies to entities within the specified country. For example, the country associated with a national code system.
Control0..1
BindingCountry: see
Typecode
Namespace.category
DefinitionCategorizes a namespace for easier search by grouping related namespaces.
Control0..1
BindingNamespaceCategory: Identifies the general purpose of the namespace identifiers provided (see http://hl7.org/fhir/namespace-category for values)
TypeCodeableConcept
CommentsThis will most commonly be used for identifier namespaces, but categories could potentially be useful for code systems and authorities as well.
Namespace.description
DefinitionDetails about what the namespace identifies including scope, granularity, etc.
Control0..1
Typestring
Namespace.usage
DefinitionProvides guidance on the use of the namespace, including the handling of formatting characters, use of upper vs. lower case, etc.
Control0..1
Typestring
Namespace.uniqueId
DefinitionIndicates how the system may be identified when referenced in electronic exchange.
Control1..*
CommentsMultiple identifiers may exist, either due to duplicate registration, regional rules, needs of different communication technologies, etc.
Namespace.uniqueId.type
DefinitionIdentifies the unique identifier scheme used for this particular identifier.
Control1..1
BindingNamespaceIdentifierType: Identifies the style of unique identifier used to identify a namepace (see http://hl7.org/fhir/namespace-identifier-type for values)
Typecode
CommentsDifferent identifier types may be used in different types of communications (OIDs for v3, URIs for FHIR, etc.).
Namespace.uniqueId.value
DefinitionThe string that should be sent over the wire to identify the code system or identifier system.
Control1..1
Typestring
Namespace.uniqueId.preferred
DefinitionIndicates whether this identifier is the "preferred" identifier of this type.
Control0..1
Typeboolean
Namespace.uniqueId.period
DefinitionIdentifies the period of time over which this identifier is considered appropriate to refer to the namespace. Outside of this window, the identifier might be non-deterministic.
Control0..1
TypePeriod
CommentsWithin a registry, a given identifier should only be "active" for a single namespace at a time. (Ideally, an identifier should only ever be associated with a single namespace across all time).
Namespace.responsible
DefinitionThe organization that is responsible for issuing identifiers or codes for this namespace and ensuring their non-collision.
Control0..1
TypeResource(Organization)
CommentsThis is the primary organization. Responsibility for some aspects of a namespace may be delegated.
Namespace.contact
DefinitionThe person who can be contacted about this system registration entry.
Control0..1
Namespace.contact.name
DefinitionNames of the person who can be contacted.
Control0..1
TypeHumanName
Namespace.contact.telecom
DefinitionIdentifies the mechanism(s) by which they can be contacted.
Control0..*
TypeContact
